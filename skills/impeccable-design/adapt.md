Modify and adjust interface for different devices, contexts, user needs, and environmental factors. Ensure optimal experience across diverse usage scenarios while maintaining core functionality and brand consistency.

**First**: Use the impeccable-design skill for responsive design principles and adaptive interfaces.

## Adaptation Framework

Comprehensive approach to creating adaptable interfaces:

1. **Device Adaptation** - Address:
   - Screen size variations (mobile, tablet, desktop, large screens)
   - Input method differences (touch, mouse, keyboard, stylus)
   - Hardware capabilities (processing power, memory, sensors)
   - Resolution and pixel density differences
   - Orientation changes (portrait, landscape)
   - Foldable and flexible screen considerations

2. **Contextual Adaptation** - Consider:
   - Usage environment (bright sunlight, dim rooms, public spaces)
   - User goals and tasks in different contexts
   - Available time and attention levels
   - Connectivity and network conditions
   - Privacy and security requirements
   - Cultural and regional preferences

3. **User Adaptation** - Accommodate:
   - Accessibility needs and assistive technologies
   - Personalization preferences
   - Language and localization requirements
   - Age-related considerations
   - Temporary or situational limitations
   - Expertise level differences

4. **Environmental Adaptation** - Respond to:
   - Network speed and reliability
   - Battery level and power management
   - Ambient lighting conditions
   - Noise levels and audio availability
   - Physical location and movement
   - Time of day and usage patterns

## Adaptation Process

### Assessment Phase
1. **Identify target contexts**: Determine all environments where interface will be used
2. **Analyze user needs**: Understand how user goals vary by context
3. **Evaluate current limitations**: Identify where current design falls short
4. **Prioritize adaptation needs**: Focus on most critical adaptation requirements
5. **Consider technical constraints**: Understand implementation limitations

### Strategy Development
1. **Define adaptation priorities**: Rank adaptation needs by importance
2. **Plan responsive behaviors**: Determine how interface should adapt
3. **Design progressive enhancement**: Ensure core functionality always works
4. **Create adaptation specifications**: Document how adaptations should work
5. **Plan testing approach**: Determine how to validate adaptations

## Adaptation Techniques

### Responsive Design
- **Fluid layouts**: Use relative units and flexible containers
- **Breakpoint strategy**: Plan logical breakpoints based on content
- **Flexible images**: Ensure images adapt to container sizes
- **Scalable typography**: Implement responsive text sizing
- **Component flexibility**: Design components that work at any size
- **Touch adaptation**: Optimize for touch interaction on small screens

### Context-Aware Features
- **Network awareness**: Adapt functionality based on connection quality
- **Location services**: Use location to enhance relevant features
- **Time awareness**: Adapt based on time of day or day of week
- **Usage patterns**: Learn from user behavior to improve experience
- **Device capabilities**: Detect and utilize available hardware features
- **Environmental sensors**: Use ambient light, motion, etc. when appropriate

### Accessibility Adaptation
- **Screen reader compatibility**: Ensure all functionality works with assistive tech
- **Keyboard navigation**: Support full functionality without mouse
- **Reduced motion**: Respect user preferences for motion reduction
- **High contrast mode**: Support accessibility themes
- **Text scaling**: Maintain usability at various text sizes
- **Alternative input**: Support various input methods and preferences

### Personalization
- **User preferences**: Remember and apply individual settings
- **Usage history**: Adapt based on past user behavior
- **Role-based interfaces**: Customize for different user types
- **Language preferences**: Support multiple languages and regions
- **Cultural adaptation**: Adjust for cultural differences and preferences
- **Expertise level**: Adapt complexity based on user familiarity

## Quality Standards

Verify that adaptations:
- **Maintain core functionality**: All essential features work in all contexts
- **Preserve brand identity**: Adaptations maintain brand consistency
- **Ensure accessibility**: All users can access adapted interfaces
- **Perform well**: Adaptations don't introduce performance issues
- **Feel native**: Interfaces feel appropriate for each context
- **Remain maintainable**: Adaptations don't overly complicate codebase

## Implementation Guidelines

### Progressive Enhancement
1. **Core functionality first**: Ensure basic features work everywhere
2. **Layer enhancements**: Add advanced features progressively
3. **Graceful degradation**: Maintain usability when features fail
4. **Feature detection**: Use modern detection techniques
5. **Performance consideration**: Optimize for slower connections/devices

### Adaptive Architecture
- **Component flexibility**: Design components that adapt naturally
- **Configuration systems**: Use configuration to drive adaptations
- **Conditional loading**: Load appropriate resources for context
- **Dynamic behavior**: Adjust behavior based on detected conditions
- **Modular design**: Keep adaptations modular and maintainable

### Testing Requirements
- **Multi-device testing**: Test on actual target devices
- **Network condition testing**: Test under various connection speeds
- **Accessibility testing**: Verify adaptations work with assistive tech
- **User testing**: Validate adaptations with real users in context
- **Performance testing**: Ensure adaptations don't hurt performance

## Context-Specific Adaptations

### Mobile Adaptations
- **Thumb-friendly targets**: Ensure touch targets are appropriately sized
- **Vertical scrolling**: Optimize for vertical scrolling patterns
- **Reduced cognitive load**: Simplify interfaces for mobile context
- **Offline capability**: Support functionality without network
- **Battery optimization**: Minimize resource usage
- **One-handed use**: Consider users operating with one hand

### Desktop Adaptations
- **Mouse and keyboard**: Optimize for precision input methods
- **Larger screen real estate**: Take advantage of available space
- **Advanced features**: Enable more complex functionality
- **Keyboard shortcuts**: Provide efficient keyboard navigation
- **Multi-window workflows**: Support complex multitasking
- **High-resolution displays**: Optimize for various pixel densities

### Accessibility Adaptations
- **Screen reader optimization**: Ensure logical content order
- **Focus management**: Maintain clear keyboard navigation
- **Alternative text**: Provide meaningful descriptions
- **Color independence**: Don't rely solely on color for meaning
- **Customizable interfaces**: Allow user adjustments
- **Alternative interaction**: Support various input methods

## Documentation Standards

Record adaptation implementations:
- **Adaptation rationale**: Why specific adaptations were implemented
- **Target contexts**: Which environments and users are supported
- **Implementation approach**: How adaptations were achieved
- **Testing methodology**: How adaptations were validated
- **Performance impact**: Any performance considerations
- **Maintenance requirements**: How to update adaptations over time

**CRITICAL**: Adaptations should enhance rather than compromise the user experience. Every adaptation should serve real user needs in specific contexts while maintaining the core value proposition of the interface.
