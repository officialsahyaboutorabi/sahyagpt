Improve unclear UX copy, confusing instructions, ambiguous labels, and vague messaging. Enhance clarity and comprehension for better user understanding and more effective user experiences.

**First**: Use the impeccable-design skill for UX writing principles and clarity standards.

## Clarity Assessment Framework

Evaluate and improve across these dimensions:

1. **Label Clarity** - Examine:
   - Button and link text that clearly indicates action
   - Form field labels that explain expected input
   - Navigation items that accurately represent destinations
   - Icon labels that provide clear meaning
   - Section headers that accurately summarize content

2. **Instructional Clarity** - Review:
   - Step-by-step guidance that's easy to follow
   - Error messages that explain problems and solutions
   - Onboarding flows that orient new users effectively
   - Help text that provides necessary context
   - Tooltips that enhance understanding

3. **Information Clarity** - Assess:
   - Data presentation that's easy to scan and understand
   - Status messages that clearly communicate system state
   - Warnings and alerts that convey urgency appropriately
   - Instructions that can be followed without ambiguity
   - Content hierarchy that guides comprehension

4. **Language Appropriateness** - Verify:
   - Vocabulary matches user expertise level
   - Tone aligns with brand and context
   - Messages are concise without sacrificing clarity
   - Language is inclusive and accessible
   - Cultural sensitivity in messaging

## Clarity Improvement Process

### Audit Current State
1. **Identify confusion points**: Where do users typically struggle?
2. **Catalog unclear elements**: List all problematic copy elements
3. **Assess user impact**: How does unclear copy affect user success?
4. **Prioritize improvements**: Focus on elements that cause most friction

### Apply Clarity Principles
1. **Be specific rather than generic**: Replace vague terms with precise language
2. **Use active voice**: Make subjects and actions clear
3. **Lead with conclusions**: Put the most important information first
4. **Use familiar terms**: Avoid jargon unless users expect it
5. **Be concise**: Say what needs to be said without excess

### Rewrite with Purpose
1. **Define the goal**: What should the user understand or do?
2. **Consider the context**: What does the user already know?
3. **Choose clear words**: Select terms that match user mental models
4. **Test for comprehension**: Would a typical user understand this?

## Clarity Techniques

### For Buttons and Actions
- **Instead of**: "Submit", "Go", "Click here"
- **Use**: "Save changes", "Send message", "View report"
- **Principle**: Make the action clear and specific

### For Error Messages
- **Instead of**: "Error occurred", "Invalid input"
- **Use**: "Password must be at least 8 characters", "Email address is required"
- **Principle**: Explain what went wrong and how to fix it

### For Instructions
- **Instead of**: "Follow the steps below"
- **Use**: "Enter your email address and click Subscribe"
- **Principle**: Be specific about required actions

### For Labels
- **Instead of**: "Info", "Details", "More"
- **Use**: "Account settings", "Billing information", "View transaction history"
- **Principle**: Describe what the user will find

## Quality Standards

Each piece of copy should meet these standards:

### Clarity
- **Unambiguous**: Cannot be misinterpreted
- **Actionable**: Tells the user what to do
- **Specific**: Uses precise, concrete language
- **Complete**: Provides necessary context

### Usability
- **Scannable**: Easy to quickly understand
- **Concise**: No unnecessary words
- **Consistent**: Uses same terms throughout
- **Accessible**: Understandable by diverse users

### Effectiveness
- **Goal-oriented**: Supports user objectives
- **Persuasive**: Encourages appropriate actions
- **Trust-building**: Uses honest, reliable language
- **Brand-aligned**: Reflects appropriate tone

## Before/After Documentation

Record improvements made:
- **Original copy**: What was unclear
- **Improved copy**: What replaced it
- **Reasoning**: Why the change improves clarity
- **Expected impact**: How this helps users

## Testing Clarity

Verify improvements by:
- **Reading aloud**: Does it sound natural and clear?
- **User perspective**: Would a first-time user understand this?
- **Task completion**: Does the copy help users achieve goals?
- **Error prevention**: Does it prevent common mistakes?

**CRITICAL**: Clarity should never be sacrificed for cleverness or branding. The primary goal is user comprehension and task completion.
