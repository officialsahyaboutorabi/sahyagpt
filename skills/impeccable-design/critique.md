Perform a comprehensive UX design review focusing on hierarchy, clarity, emotional resonance, and overall user experience quality. Evaluate the interface from a user-centered perspective and provide constructive feedback for improvement.

**First**: Use the impeccable-design skill for design principles and anti-patterns.

## UX Review Framework

Evaluate across multiple UX dimensions:

1. **Visual Hierarchy** - Assess:
   - Information architecture and content prioritization
   - Clear pathways for user goals
   - Proper emphasis through size, color, and positioning
   - Logical flow and scannability
   - Focal points that guide attention appropriately

2. **Clarity & Usability** - Check:
   - Clear affordances for interactive elements
   - Understandable labels and instructions
   - Consistent patterns and mental models
   - Intuitive navigation and information scent
   - Error prevention and recovery mechanisms

3. **Emotional Resonance** - Evaluate:
   - Brand alignment and personality expression
   - Appropriate tone for target audience
   - Delightful micro-interactions and feedback
   - Confidence-inspiring design decisions
   - Trust-building elements and credibility signals

4. **Cognitive Load** - Analyze:
   - Mental effort required to complete tasks
   - Information density and organization
   - Decision points and choice architecture
   - Memory requirements for task completion
   - Distractions and irrelevant elements

5. **Accessibility & Inclusion** - Consider:
   - Universal design principles
   - Diverse user needs and abilities
   - Cultural sensitivity and localization
   - Multiple input methods and assistive technologies
   - Bias prevention in design decisions

## Critique Structure

Provide feedback in this structured format:

### Executive Assessment
- Overall UX quality rating (1-10 scale)
- Primary strengths and weaknesses
- Critical issues requiring immediate attention
- Opportunities for significant improvement

### Strengths Analysis
Highlight what's working well:
- Effective design decisions that serve users
- Strong visual hierarchy and information architecture
- Well-executed interactions and feedback
- Successful emotional design elements
- Accessibility considerations that work

### Areas for Improvement
Organized by impact level:

#### Critical Issues
[Problems that significantly impede user goals or create major frustration]

#### High-Impact Improvements
[Changes that would substantially improve user experience]

#### Moderate Enhancements
[Refinements that would improve the experience]

#### Minor Optimizations
[Small tweaks for polish]

### Specific Recommendations
For each issue, provide:
- **Problem**: What's not working
- **Impact**: How it affects users
- **Solution**: Specific recommendation for improvement
- **Priority**: How urgent this is to address
- **Alternative**: Other approaches to consider

### Design Principles Alignment
Assess how well the design follows key principles:
- Does it prioritize user goals over business objectives?
- Is it consistent with the brand and context?
- Does it minimize cognitive load?
- Does it provide clear feedback and affordances?
- Is it inclusive and accessible?

### Iteration Suggestions
Recommend specific experiments or A/B tests to validate improvements, and suggest metrics to measure success.

**CRITICAL**: Balance constructive criticism with appreciation for what works well. Frame feedback positively and provide actionable next steps. Remember that design is iterative - focus on improvement rather than perfection.
