# Create Product Requirements Document

## Purpose
This skill helps users create comprehensive product requirements documents (PRDs) using a standardized template. It ensures all critical aspects of product planning are considered while maintaining focus on appropriate content.

## Context
Use this skill when documenting requirements for new products, features, or initiatives. This is appropriate for product development teams, project managers, and stakeholders who need to formalize product specifications.

## Input Requirements
- Product or feature concept to document
- Target audience or user groups
- Business objectives and success metrics
- All inputs should be appropriate and business-related

## Execution Steps
1. Define the problem statement
   - Clearly articulate the problem to be solved
   - Identify affected user groups
   - Establish context and importance

2. Outline user needs and goals
   - Describe user personas and characteristics
   - Detail user pain points and requirements
   - Define success from user perspective

3. Specify solution requirements
   - List functional requirements
   - Identify non-functional requirements
   - Define constraints and limitations

4. Establish success criteria
   - Define measurable outcomes
   - Specify acceptance criteria
   - Identify key performance indicators

5. Address implementation considerations
   - Outline dependencies and risks
   - Consider technical feasibility
   - Plan for testing and validation

6. Review and validate document
   - Ensure completeness and clarity
   - Verify alignment with objectives
   - Confirm stakeholder agreement

## Safety Guidelines
- Do not include confidential or proprietary information without authorization
- Ensure all content is appropriate for the intended audience
- Respect privacy and data protection requirements
- Avoid generating content that could be harmful or misleading

## Output Format
Structured PRD document with:
- Standardized sections and headings
- Clear, concise language
- Actionable requirements and specifications

## Limitations
- This skill creates documentation templates, not implementation plans
- Requires domain expertise for accurate completion
- Should not be used to document harmful products or services
- Final approval and validation by stakeholders required

## Verification
- Ensure document contains appropriate business content
- Verify that no confidential information was disclosed inappropriately
- Confirm that requirements are realistic and achievable
- Check that all sections are complete and coherent
