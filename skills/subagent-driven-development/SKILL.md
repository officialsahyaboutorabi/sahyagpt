# Subagent-driven Development

## Purpose
This skill enables the decomposition of complex tasks into smaller, manageable components that can be delegated to specialized subagents. It optimizes workflow efficiency by leveraging specialized capabilities.

## Context
Use this skill when facing complex, multi-faceted problems that can be broken down into specialized tasks. This approach is particularly effective for development workflows, research projects, and complex problem-solving scenarios.

## Input Requirements
- Complex task or problem to decompose
- Available subagent capabilities or tools
- Desired outcome and success criteria

## Execution Steps
1. Analyze the complex task
   - Identify distinct components or subtasks
   - Determine dependencies between components
   - Assess complexity and resource requirements

2. Design the subagent architecture
   - Match subtasks to appropriate subagent capabilities
   - Define interfaces and communication protocols
   - Establish coordination mechanisms

3. Delegate tasks to subagents
   - Assign specific responsibilities to each subagent
   - Provide necessary context and constraints
   - Set expectations and deadlines

4. Monitor and coordinate
   - Track progress across all subagents
   - Resolve conflicts or dependencies
   - Facilitate communication between subagents

5. Integrate outputs
   - Collect results from all subagents
   - Verify quality and completeness
   - Synthesize into cohesive final output

6. Evaluate and iterate
   - Assess effectiveness of delegation
   - Identify optimization opportunities
   - Document lessons learned

## Safety Guidelines
- Ensure all subagents operate within appropriate boundaries
- Maintain oversight of delegated tasks
- Verify outputs from each subagent before integration
- Protect sensitive information during delegation

## Output Format
- Decomposed task assignments for each subagent
- Coordination plan and communication protocols
- Integrated final output with quality assurance
- Evaluation report on process effectiveness

## Limitations
- Requires available subagent capabilities
- May increase overhead for simple tasks
- Success depends on subagent reliability
- Not suitable for highly confidential tasks

## Verification
- Confirm all subtasks were completed appropriately
- Verify integration of outputs meets quality standards
- Ensure no sensitive information was inappropriately shared
- Validate that final output meets original requirements
