# UI Developer Skill

Composite skill for creating and revising user interfaces. This skill integrates multiple specialized capabilities to deliver distinctive, production-ready UIs with exceptional design quality and technical excellence.

## Skill Composition

This skill combines:
- **System Design** - Architectural planning and integration
- **Impeccable Design** (16 commands) - Visual and UX excellence
- **React Best Practices** - Implementation patterns
- **Subagent-driven Development** - Parallel task execution
- **Test-driven Development** - Quality assurance
- **Brainstorming** - Creative ideation
- **Writing Skills** - Documentation and UX copy

---

## When to Use

Activate this skill when:
- Creating new user interfaces from scratch
- Revising or redesigning existing interfaces
- Integrating multiple platforms into unified UI
- Adding new features to existing UI
- Improving UI/UX quality and consistency
- Creating component libraries or design systems

## Development Phases

### Phase 1: Discovery & Planning

**Objective:** Understand requirements and architect the solution

**Activities:**
1. **Context Gathering**
   - Target audience identification
   - Use case analysis
   - Technical constraints assessment
   - Brand personality definition

2. **System Architecture**
   - Component hierarchy design
   - State management strategy
   - API integration planning
   - Data flow mapping

3. **Creative Ideation**
   - Navigation structure brainstorming
   - Feature integration concepts
   - Visual design directions
   - User workflow optimization

**Deliverables:**
- Architecture diagram
- Component tree
- Data flow documentation
- User journey maps

---

### Phase 2: Design System

**Objective:** Establish visual foundation and component library

**Activities:**
1. **Visual Foundation** (Colorize)
   - Color palette definition
   - Theme system (light/dark)
   - Brand color integration
   - Accessibility contrast validation

2. **Design Tokens** (Normalize)
   - Typography scale
   - Spacing system
   - Border radius values
   - Shadow tokens
   - Opacity values

3. **Component Library** (Extract)
   - Atomic components (buttons, inputs, etc.)
   - Molecule components (forms, cards, etc.)
   - Organism components (headers, modals, etc.)
   - Template components (layouts, pages, etc.)

**Deliverables:**
- Design token system
- Component library structure
- Style guide documentation

---

### Phase 3: Implementation

**Objective:** Build functional components with quality standards

**Activities:**
1. **Component Development** (React Best Practices)
   - Functional components with hooks
   - Proper prop typing
   - Memoization where appropriate
   - Lazy loading for code splitting

2. **Parallel Development** (Subagent-driven)
   - Multiple components simultaneously
   - API integration in parallel
   - Concurrent feature development
   - Distributed testing

3. **Test Coverage** (TDD)
   - Unit tests for utilities and hooks
   - Component tests with rendering
   - Integration tests for workflows
   - E2E tests for critical paths

**Deliverables:**
- Functional components
- Test suites
- Integration code

---

### Phase 4: Refinement

**Objective:** Polish and enhance the user experience

**Activities:**
1. **Visual Polish** (Polish)
   - Pixel-perfect alignment
   - Typography refinement
   - Color enhancement
   - Animation smoothing

2. **Micro-interactions** (Delight)
   - Hover states
   - Loading animations
   - Success/error feedback
   - Transitional effects

3. **Motion Design** (Animate)
   - Purposeful animations
   - Proper easing curves
   - Performance optimization (60fps)
   - Reduced motion support

4. **Resilience** (Harden)
   - Error boundaries
   - Edge case handling
   - Loading states
   - Offline support

**Deliverables:**
- Polished UI
- Animation system
- Error handling

---

### Phase 5: Launch Preparation

**Objective:** Prepare for production deployment

**Activities:**
1. **User Onboarding** (Onboard)
   - First-time user flows
   - Feature tutorials
   - Contextual help
   - Empty states

2. **UX Copy** (Clarify)
   - Button labels
   - Error messages
   - Help text
   - Microcopy refinement

3. **Documentation** (Writing Skills)
   - Component documentation
   - Usage examples
   - API integration guides
   - Developer handoff docs

4. **Final Audit** (Audit)
   - Accessibility review
   - Performance audit
   - Cross-browser testing
   - Responsive verification

**Deliverables:**
- Onboarding flows
- Complete documentation
- Audit report

## Anti-Patterns (What to Avoid)

### Visual Anti-Patterns
- ✗ Overused fonts (Inter, Roboto, Arial, system defaults)
- ✗ Gray text on colored backgrounds
- ✗ Pure black (#000) or pure white (#fff) without tinting
- ✗ Everything wrapped in cards or nested cards
- ✗ Identical card grids repeated endlessly
- ✗ Gradient text for "impact"
- ✗ Glassmorphism everywhere
- ✗ Bounce/elastic easing

### Technical Anti-Patterns
- ✗ Animating layout properties (use transform/opacity)
- ✗ Missing accessibility attributes
- ✗ No loading or error states
- ✗ Hardcoded values instead of tokens
- ✗ Missing test coverage
- ✗ Poor component separation

### UX Anti-Patterns
- ✗ Unclear button labels ("Submit", "Click here")
- ✗ Redundant information
- ✗ Missing feedback for user actions
- ✗ Inconsistent interaction patterns
- ✗ No empty state design
- ✗ Ignoring reduced motion preferences
